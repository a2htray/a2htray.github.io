Golang Import cycle not allowed

Posted on 2020-08-06

开发时，遇到报错import cycle not allowed，出现该问题的原因在于 Go 不支持循环引入包。

在项目中，存在以下 3 个包：

parent/model：定义与数据库表相关的模型

parent/service：对外提供的服务实现

parent/store：具体数据库 CURD 操作

其中 service 引入了 store， store 引入了 model，然后又想在 model 中引入 store。简单的想法是希望在model 中定义一些用户操作，如 login 和 logout。正是因为这样，store 和 model 存在相互引用的问题。出于解决问题的角度，model 不再引入 store，而是在 service 中实现用户的 login 和 logout。

或许有更好的组织结构的方式。
