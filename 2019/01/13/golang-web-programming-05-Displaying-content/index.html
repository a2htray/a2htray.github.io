模板引擎的介绍

Web template engines are an important part of any web application framework, and most if not all full-fledged frameworks have one. 模板引擎很重要

Templates and template engines

Web template engines evolved from server-side includes (SSIs), which eventually branched out into web programming languages like PHP, ColdFusion, and JSP. 模板引擎的历史there are roughly two ideal types of template engines, at opposite ends of the spectrum:两种意识形态：是否包含业务逻辑

  • Logic-less template engines: Dynamic data is substituted into the templates at specified placeholders. 处理好的数据填充到占位符
  • Embedded logic template engines: Programming language code is embedded into the template and executed during runtime by the template engine, including substitutions. 内嵌程序代码

Logic-less template engines are usually faster to render because less processing is involved. 微逻辑模板引擎速度更快点Go’s template engine, mostly in text/template and the HTML-specific bits in html/template, is such a hybrid.Go语言的模板引擎是两种都包含的混合体

The Go template engine

In a web app, the handler normally triggers the template engine. The template engine then generates the HTML and writes it to the ResponseWriter, which adds it to the HTTP response sent back to the client.模板引擎生成HTML代码通过ResponseWriter返回给客户端

In the Go template engine, a template is text that has embedded actions. The text is parsed and executed by the template engine to produce another piece of text.模板文件是文本形式的，解析模板文本生成HTML

text/template 用于任何形式的文本

html/template 用于HTML

Actions are added between two double braces, {\{ and }\}.

Template files must be in a readable text format but can have any extension.必须可读，但不限扩展名.The dot is an action, and it’s a command for the template engine to replace it with a value when the template is executed. (.)表示有数据填充

两步：

  • Parse the text-formatted template source, which can be a string or from a template file, to create a parsed template struct.将模板文件解析成template结构
  • Execute the parsed template, passing a ResponseWriter and some data to it.

Parsing templates

Another way to parse files is to use the ParseGlob function, which uses pattern matching instead of specific files.

In fact, all other ways of parsing templates ultimately call the Parse method to parse the template. 所有的解析操作内部都是调用Parse方法解析字符串

Executing templates

The usual way to execute a template is to call the Execute function on a template, passing it the ResponseWriter and the data. 调用Execute方法，并将结果传递给ResponseWriter

Include actions

模板包含模板操作、template

allow us to include a template in another template.

we need to parse both the template files that we’re going to use. 需要解析用到的两个模板文件，这个非常重要!

there’s a variant of the include action that allows this: {\{ template “name” arg }\} , where arg is the argument you want to pass on to the included template. 向内嵌的模板传入参数

Arguments, variables, and pipelines

An argument is a value that’s used in a template. It can be a Boolean, integer, string, and so on. It can also be a struct, or a field of a struct, or the key of an array.

We can also set variables in the action. Variables start with the dollar sign ($) and look like this:在action中以$开头

Pipelines are arguments, functions, and methods chained together in a sequence.

Functions

自定义模板函数

To define custom functions, you need to:

  • Create a FuncMap map, which has the name of the function as the key and the actual function as the value.
  • Attach the FuncMap to the template.

Context awareness

根据模板中参数的所在位置进行相应的编码

The content that’s displayed changes depending on where you place it within the document; that is, the display of the content is context-aware.

this feature can be used to defend against XSS (cross-site scripting) attacks.

Defending against XSS attacks

显示了用户提供的数据

向网页中注入了其他用户不可见的代码

A common XSS attack is the persistent XSS vulnerability. This happens when data provided by an attacker is saved to the server and then displayed to other users as it is.

malicious code is executed with the user’s permissions but without the user’s knowledge.

Go allows us to escape from being context-aware if we really want to.

Unescaping HTML

Go provides a mechanism to “unescape” HTML. Just cast your input string to template.HTML and use that instead, and our code is happily unescaped.

If you really want to stop the browser from protecting you from XSS attacks, you simply need to set a response header in our handler:通过设置Header头，关闭浏览器的XSS保护

Nesting templates

布局、

Show you how you can use layouts in your web app.

Layouts are fixed patterns in web page design that can be reused for multiple pages.

Notice that we start the file with {\{ define “layout” }\} and end it with {\{ end }\}. Anything within these two action tags is considered part of the layout template.

Using the block action to define default templates

block的作用在于是否传递了要在layout中显示的内容，如果没有则显示默认的

Allows you to define a template and use it at the same time.

The block action effectively defines a template named content and also places it in the layout. If no content template is available when the overall template is executed, the content template defined by the block will be used instead.

Summary

  • Go’s standard template engine is in the html/template package.
  • Actions are instructions about how data is to be combined with the template. Go has an extensive and powerful set of actions
  • Customized functions can also be created by making a function map and attaching it to the template.
