Introducing web services

Web services, at a basic level, are software programs that interact with other software programs. 与其他软件程序交互的软件程序

there are different types of web services, including SOAP-based, REST-based, and XML-RPCbased. 不同类型的 Web 服务

  • SOAP-based：多运用于企业内部
  • REST-based：对外
  • XML-RPCbased

SOAP-based web services are mostly being used in enterprise systems; REST-based web services are more popular in publicly available web services.

SOAP-based services are robust, are explicitly described using WSDL (Web Service Definition Language), and have built-in error handling. Together with UDDI (Universal Description, Discovery, and Integration—a directory service), SOAP-based services can also be discovered

  • WSDL：Web Service Definition Language
  • UDDI：Universal Description, Discovery, and Integration

REST-based web services are a lot more flexible. REST isn’t an architecture in itself but a design philosophy. It doesn’t require XML, and very often REST-based web services use simpler data formats like JSON, resulting in speedier web services. REST-based web services are often much simpler than SOAP-based ones.REST-based比SOAP-based简单,常使用JSON作为传输格式

Another difference between the two is that SOAP-based web services are function-driven; REST-based web services are data-driven.SOAP是功能驱动、REST是数据驱动

SOAP-based web services tend to be RPC (Remote Procedure Call)远程程序调用 RPC styled; REST-based web services, as described earlier, focus on resources, and HTTP methods are the verbs working on those resources.SOAP-based类似于远程程序的调用，而HTTP类似于资源的获取

SOAP-based与REST-based区别在于

  • 定义明确
  • 以功能为驱动
  • 设计过重
  • 结构性

SOAP is highly structured and heavily defined, and the XML used for the transportation of the data can be complex. Parsing and creating XML with Go

look at how Go can be used to create and consume them.

XML is a popular markup language (HTML is another example of a markup language)标记性语言 that’s used to represent data in a structured way.结构化展示数据XML is a formal recommendation from the W3C, and it’s defined by W3C’s XML 1.0 specification.规范化

RSS：XML-based

To manipulate XML or JSON, you can use the corresponding XML or JSON subpackages of the encoding library. For XML, it’s in the encoding/xml library.

Parsing XML

In Go, you parse the XML into structs, which you can subsequently extract the data from.

Struct tags结构标签 are strings after each field that are a key-value pair. The key is a string and must not have a space, a quote ("), or a colon (:). The value must be a string between double quotes (""). For XML, the key must always be xml.

strings in Go are created using the double quotes (") and backticks (`).

the struct and all the fields in the struct that you create must be public, which means the names need to be capitalized.字段属性必须是公开的

Here are some rules for the XML struct tags:

  • 保存标签名 To store the name of the XML element itself (normally the name of the struct is the name of the element), add a field named XMLName with the type xml.Name. The name of the element will be stored in that field.
  • 保存标签的属性 To store the attribute of an XML element, define a field with the same name as that attribute and use the struct tag xml:"<name>,attr", where is the name of the XML attribute.
  • 保存标签内字符数据 To store the character data value of the XML element, define a field with the same name as the XML element tag, and use the struct tag xml:",chardata".
  • 保存未被处理的标签内数据To get the raw XML from the XML element, define a field (using any name) and use the struct tag xml:",innerxml".
  • 在没有指定结构标签的情况下 If there are no mode flags (like ,attr or ,chardata or ,innerxml) the struct field will be matched with an XML element that has the same name as the struct’s name.
  • 不需要全文转换成树的情况下 To get to an XML element directly without specifying the tree structure to get to that element, use the struct tag xml:"a>b>c", where a and b are the intermediate elements and c is the node that you want to get to.

Parsing and creating JSON with Go

JavaScript Serialized Object Notation (JSON) is a lightweight, text-based data format based on JavaScript. The main idea behind JSON is that it’s easily read by both humans and machines.人和机器都能很好的识别 JSON is popularly used in REST-based web services, although they don’t necessarily need to accept or return JSON data.

Go is also used to create web services for Internet of Things (IoT) and wearables such as smart watches.说明下应用场景

Go’s support for JSON is from the encoding/json library.

Parsing JSON

  • Create structs to contain the JSON data.
  • Use json.Unmarshal to unmarshal the JSON data into the structs

Creating JSON

Creating JSON is also is similar to creating XML

Summary

  • There are mainly two types of web services: SOAP-based and REST-based web services
  • Creating and parsing XML and JSON are similar and involve creating a struct and either generating (unmarshaling) XML or JSON from it, or creating a struct and extracting (marshaling) XML or JSON into it.
