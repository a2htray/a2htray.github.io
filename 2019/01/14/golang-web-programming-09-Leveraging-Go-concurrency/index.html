介绍 concurrency 和讨论 Go 中 concurrency 的模型及设计

讨论两种由 Go 提供的 concurrency 特性: goroutines 和 channels

Concurrency isn’t parallelism

Concurrency 是指两个或多个任务开始运行，并在相同的时间段落内结束，这些任务相互之间进行协作的机制。任务是相对于其他任务是同时存在的，而不是按顺序执行的

Concurrency 与 Parallelism 的不同

Parallelism 通常需要独立的资源，如 CPU; concurrency 则是使用相同的资源

Goroutines

独立的

Goroutines 是与其他 goroutines 独立运行的功能，是基于线程 threads，但它们不是线程

Summary

  • The Go web server itself is concurrent and each request to the server runs on a separate goroutine.
  • Concurrency is not parallelism
  • Go supports concurrency directly with two main features: goroutines and channels. Go doesn’t support parallelism directly.
  • Goroutines are used to write concurrent programs, whereas channels are used to provide the communications between goroutines.两者用途不同
  • A select statement can be used to choose one out of a number of channels—the first channel that is ready to be received will be selected
  • Concurrent web applications can be highly performant, depending on the algorithm used, even on a single CPU, compared with non-concurrent web applications.算法好，单核也 NB
