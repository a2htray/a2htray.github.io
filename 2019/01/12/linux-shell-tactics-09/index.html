Linux -Linux Shell 脚本攻略- 09 管理重任

Posted on 2019-01-12 Edited on 2020-08-23

GUN/Linux 的生态系统是由运行的程序、服务、连接的设备、文件系统、用户等组成，需要对各类信息进行管理、监控

收集进程信息 top、ps、pgrep

进程是程序的运行实例，运行在一台计算机中的多个进程各自都分配了一个称为进程ID的唯一标识写数字

ps 命令中的 -f 选项，可以显示更多列的信息

  • -e 或 -ax：显示所有，e 表示every，ax 表示all

结合 -e 和 -f，显示所有的进程信息及各进程的列信息

  • -o：指定需要显示的列，要显示的列以逗号作为分隔符

-o 可以指定的列名

参数 描述
pcpu CPU占用率
pid 进程ID
ppid 父进程ID
pmem 内存使用率
comm 可执行文件名
cmd 简单命令
user 启动进行的用户
nice 优先级
time 累计的CPU时间
etime 进程启动后度过的时间
tty 所关联的TTY设备
euid 有效用户ID
stat 进程状态
  • --sort：根据参数对按特定列进行排序，+ 升序，- 降序

pgrep：获得特定命令的进程ID列表

  • -u：指定特定用户
  • -c：返回所匹配的进程数量

杀死进程以及发送或响应信号

在类UNIX系统中与进程相关的一个重要概念就是信号，信号是一种进程间通信的机制，它用来中断运行的进程以执行某些操作。我们可以使用特定的信号来中断进程

通过 kill 命令向进程发送指定的信号

SIGNAL 是信号名称或者信号数，常用的如下：

信号名称 信号数 描述
SIGHUP 1 对控制进程或终端进行挂起检测
SIGINT 2 当按下Ctrl+C时发送该信号
SIGKILL 9 用于强行杀死进程
SIGTERM 15 默认用于终止进程
SIGTSTP 20 当按下Ctrl+Z时发送该信号

killall通过命令名终止进程

  • -u：指定所属用户
  • -i：显示确认信息，输入y/n

pkill 命令与 kill 命令类似，不过默认情况下 pkill 接受的是进程名，而非 进程ID，如

trap 捕捉并响应信号

trap 命令在脚本中用来为信号分配信号处理程序，一旦使用 trap 将某个函数分配给一个信号，那么当脚本运行时收到这个信号，对应于信号的函数就会开始执行

对于信号处理程序而言，程序必须始终运行

如果没有为信号指定信号处理程序，那么将会调用操作系统默认分配的信号处理程序

which、whereis、file、whatis 与平均负载

which 命令用来找出某个命令的位置，而位置由环境变量 PATH 而定

whereis 命令与 which 类似，但它不仅返回命令的路径，还能够打印出其对应的命令手册的位置以及命令源代码的路径

file 命令用来确定文件的类型

whatis 命令会输出作为参数的命令的简短描述信息

平均负载由三个值确定，第一个是1分钟内，第二个是5分钟内，第三个是15分钟内

向用户终端发送信息

系统管理员可以向网络中所有主机上的所有用户或特定用户的终端发送消息

wall 命令用来向所有当前登录用户的终端写入消息。它可以将消息传递给一台服务器中所有的当前登录用户或是多台分散主机中的用户。

在Linux系统中，终端是作为设备存在的，那些打开的终端在 /dev/pts/ 中都会有对应的设备节点文件。向特定的设备写入数据将会在对应的终端中显示出消息

收集系统信息

各种系统信息包括主机名、内核版本、Linux发行版名称、CPU信息、内存信息、磁盘分区等

用 /proc 收集信息

/proc是一个位于内存中的伪文件系统，它的引入是为了提供一个可以从用户空间读取系统参数的接口

系统中每一个运行的进程在/proc中都有一个对应的目录，进程的目录名和进程ID相同

用 cron 进行调度

守护进程 cron 使得任务能够以固定的时间间隔在系统后台自动运行。cron 利用的是一个被称为 cron表 的文件，这个文件中存储了需要执行的脚本或命令的调度列表以及执行时间。

只要我们在 cron表 中写入条目，对应的命令就会在指定的时间执行。命令 crontab 用来添加调度条目。每一个用户都有自己的 cron 调度，通常这也被称为一个 cron作业

cron表 的每一个条目都是由6部分组成，并按照下列顺序排列

  • 分钟(0-59)
  • 小时(0-23)
  • 天(1-31)
  • 月份(1-12)
  • 工作日(0-6)
  • 命令(指定时间需要执行的命令)

* 指定命令应该在每一个时间阶段执行，如果希望在某个特定时段执行命令，那么就在对应的时间字段中指定时段，并用逗号分隔。还有另一个不错的选项可以让我们以特定的时间间隔运行命令。在分钟字段使用 */5，可以每5分钟运行一次命令
