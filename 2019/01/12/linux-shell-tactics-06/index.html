压缩及解压缩操作，tar

用 tar 进行归档

格式：tar 选项 包名 文件列表

tar命令用于归档文件，可以将多个文件或目录保存成单个文件，同时还能保留所有的文件属性

A c d r u x f v

  • -c：表示创建文件
  • -f：指定创建文件的文件名
  • -v或-vv：显示具体信息

如果有需要创建文件，则f必为最后一项

  • -r：追加文件
  • -x：用于提取文件
  • -C：指定提取到的目录
  • -A：合并两个tar包
  • -t：查看tar包内容
  • -u：通过检查时间戳来更新归档文件中的内容，只比归档文件中内容新才被加进去
  • --delete：从给定的归档文件中删除文件

不同的tar选项可以用来指定不同的压缩格式

  • -j：指定bunzip2格式
  • -z：指定gzip格式
  • --lzma：指定lzma格式
  • -a或--auto-compress：根据文件文件名自动识别压缩格式
  • --exclude：通过指定样式排除匹配通配符的文件
  • -X：若文件列表记录在某文件中，可使用该选项对记录的文件进行排除
  • --exclude-cvs：不对用于管理版本的目录进行归档

例子

用 gunzip 或 gzip 压缩

gzip只能压缩单个文件，无法对目录和多个文件进行压缩

  • -l：列出压缩文件属性
  • -c：利用 stdout 进行压缩，与重定向 > 配合
  • --fast 或 --best：指定最低或最高压缩比

zcat命令无需解压缩，直接从.gz文件中提取内容，并打印到stdout

用 bunzip 或 bzip 压缩

bunzip2 是另一种与 gzip 类似的压缩技术，生成的文件比 gzip 更小

  • -c：利用 stdout 进行压缩，与重定向 > 配合
  • -k：不进行文件删除的操作

使用lzma压缩

与 gzip 和 bzip2 相比，lzma是一个相对较新的压缩工具，它提供了比 gzip 和 bzip2 更好的压缩率

lzma 会删除原文件，使用 -c 配合定向符 > 保留原文件

两种创建 lzma 压缩文件的方法，只使用 tar 和配合使用 tar

  • -k：不进行文件删除的操作

用 zip 进行压缩

完成压缩和解压缩不会删除文件

  • -r：进行递归操作
  • -u：用于更新文件操作
  • -d：从压缩文件中删除文件
  • -l：显示压缩文件中的内容

用 rsync 备份系统快照

备份数据是多数系统管理员的日常工作。我们可能要备份 Web服务器 上的数据，也可能要备份远端数据。rsync 可以对位于不同位置的文件和目录进行备份，它可以借助差异计算以及压缩技术来最小化数据传输量。

它还支持网络数据传输，在进行复制的同时，rsync 会比较源端和目的端的文件，只有文件有更新时才进行复制

使用 SSH 连接远程主机

  • -a：进行归档
  • -v：显示详细信息
  • -z：传输时进行压缩

路径信息可以是本地路径，也可以是远端服务器路径

  • --exclude：排除特定文件，利用通配符
  • --delete：删除在源端已经不存在的文件
