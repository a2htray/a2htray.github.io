Linux -Linux Shell 脚本攻略- 02 命令之乐

Posted on 2019-01-12 Edited on 2020-08-23

cat、find、xargs、tr、md5sum、sort、uniq、split、%、#

  • cat：读取、显示或拼接文件内容
  • find：文件查找
  • xargs：使用 stdin 并提供命令行参数来执行其他命令
  • tr：替换、删除以及压缩操作 translate
  • md5sum、md5deep：用于文件完整性测试的特定密钥就被称为校验和
  • sort：对文件和标准输出进行排序操作
  • uniq：唯一行操作
  • split：文件分割操作

cat

cat 命令简约却不失优美，通常用于读取、显示或拼接文件内容，还能从标准输入中读取

- 作为标准输入的文件名，将 ls -al 的标准输出与 file.txt 中内容拼接起来

选项

  • -s ：压缩连续的空白行成一个空白行
  • -n ：打印行号
  • -T：显示制表符
  • -c-N：提出每行前N个字符

find

过滤条件包括：文件名、路径名、目录深度、文件类型、文件时间、文件大小等

格式：find 目录 [选项]

find 命令的工作方式如下：沿着文件层次结构向下遍历，匹配符合条件的文件，并执行相应的操作

  • -print：使用 \n 连接条目
  • -print0：使用 \0 连接条目
  • -name：用于指定文件名必须匹配的字符串，通配符
  • -iname：与 -name 一样，但会忽略大小写，通配符
  • 使用 -o 进行或条件匹配
  • 使用 -a 进行与条件匹配
  • 使用 -path 可以对路径和文件进行匹配，使用通配符，-name 只对文件名有效
  • 使用 -regex 基于正则进行匹配
  • 使用 -iregex 基于正则进行匹配，忽略大小写
  • 使用 ! 进行否定
  • 使用 -maxdepth、-mindepth 选项限制遍历的深度
  • -type：指定文件类型
文件类型 类型参数
普通文件 f
符号链接 l
目录 d
字符设备 c
块设备 b
套接字 s
Filo p
  • 使用 -atime，用户最近一次访问文件的时间
  • 使用 -mtime，文件内容最后一次修改的时间
  • 使用 -ctime，文件元数据最后一次改变的时间

文件元数据包括：权限或所有权

上述三个选项，基于的时间单位为天，还有基于秒的，如下

  • 使用 -amin，用户最近一次访问文件的时间
  • 使用 -mmin，文件内容最后一次修改的时间
  • 使用 -cmin，文件元数据最后一次改变的时间

+ 表示大于，- 表示小于

  • 使用 -newer 选项，指定一个文件作为比较时间
  • 使用 -size 选项，过滤文件大小，+ 大于，- 小于
单位 说明
b 块(512字节)
c 字节
w 字(2字节)
k 千字节
M 兆字节
G 吉字节
  • -delete：用于删除匹配到的文件
  • -perm：基于文件权限进行匹配
  • -user：指定用户
  • -prune：跳过特定的文件或目录
  • -d: 执行深度优先查找并显示

xargs

xargs是构建单行命令的重要组件之一

xargs命令应该紧跟在管道操作符之后，它以标准输入作为主要的源数据流，并使用 stdin 并提供命令行参数来执行其他命令

格式：command | xargs

  • 将多行输入转换成单行输出
  • 将单行转换成多行输出，-n 每行最大个数

xargs 有一个选项 -I，指定一个替换字符串

  • 结合 find 使用 xargs，用于删除文件
  • 结合 find 使用 xargs，统计代码行数

tr

tr(translate) 可以对来自标准输入的字符进行替换、删除以及压缩，只能通过stdin

通过在 tr 中使用集合的概念，我们可以轻松地将字符从一个集合映射到另一个集合

  • 将输入字符由大写转换成小写
  • 进行数字加密和解密
  • 完成替换功能
  • 完成删除功能 -d
  • -c 定义一个补集
  • 使用 tr -s 进行压缩字符

使用到的字符类

字符类 说明

alpha 字母
cntrl 控制字符
digit 数字
graph 图形字符
lower 小写字母
print 可打印字符
punct 标点符号
space 空白字符
upper 大写字符
xdigit 十六进制字符

md5sum、md5deep

用于文件完整性测试的特定密钥就被称为校验和

有 md5sum、sha1sum

对目录进行校验，使用 md5deep

sort、uniq、

sort 命令可以帮助我们对文件和标准输出进行排序操作

uniq 命令通常与 sort 命令一同使用，作用是从文本或stdin中提取单一的行

sort，选项可配合使用，如果要明确采用数字顺序排序，应该明确指定 -n

  • -n 按数字进行排序
  • -r 按逆序进行排序
  • -C 用于检测文件是否以排序
    • -C 是否排序
    • -nC 是否按数字排序
  • -k 多列时，指定k值，按某列进行排序
  • -b忽略文件中的前导空白字符
  • -d 用于指明以字典序进行排序

uniq，通过消除重复内容，从给定输入中找出单一的行，也可以用来找输入中出现的重复行，只能用于排过序的数据输入

  • -u 只显示唯一的行
  • -c 显示出现的次数
  • -d 找出重复的行
  • 结合 -s 和 -w 来指定键
    • -s 指定可以跳过前N个字符
    • -w 指定用于比较的最大字符(Mac中没有)

split

  • -b 指定文件大小
  • -a 指定后缀长度
  • 命令最后一个参数指定前缀
  • -l按行数进行分割

csplit，基于文件内容进行分割，能根据指定的条件和字符串匹配选项对文件进行分割

  • /SERVER/：用来匹配某一行，分割过程从该行开始
  • {*} 匹配重复分割的次数，* 表示到文件结尾，{整数} 匹配的次数
  • -s 静默模式，不打印信息
  • -n 指定分割后数字的个数
  • -f 指定前缀
  • -b 指定后缀格式

提取文件名，或扩展名

%、#、%%、## 均是用于删除匹配结果

借助 % 或 # 操作符，可以将名称部分从 名称.扩展名 格式中提取文件名或扩展名

${VAR%.*}

  • % 用于删除右边通配符匹配的字符，非贪婪模式，匹配最短的，从左到右进行匹配
  • %% 贪婪模式，匹配最长的，从右到左进行匹配

${VAR#*.}

  • 删除位于 # 右侧通配符匹配项，从左到右
  • ##用于贪婪匹配
