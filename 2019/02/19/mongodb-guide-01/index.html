MongoDB 01 简介

Posted on 2019-02-19 Edited on 2020-08-23

MongoDB 是一款强大、灵活，且易于扩展的通用型数据库，扩展出的功能包括：

  • 二级索引 (secondary index)
  • 范围查询 (range query)
  • 排序 (sort)
  • 聚合 (aggregation)
  • 地理空间索引 (geospatial index)

易于使用

MongoDB 是 面向文档 (document-oriented) 的数据库，不是关联型数据库，即非关系数据库

面向文档型的数据库没有 行 的概念，取而代之的是 文档模型。不再有 预定义模式 (predefined schema)：文档的键 (key) 和值 (value) 不再是固定的类型和大小，即 free-schema

  • 易于添加或删除字段

易于扩展

应该如何扩展数据库？

  • 纵向扩展：使用更好的机子
  • 横向扩展：通过分区将数据分散到更多机器上，使用多个 MongoDB 实例

MongoDB 的设计采用横向扩展，能自动处理跨集群的数据和负载，自动重新分配文档，以及将用户请求路由到正确的机器上。

丰富的功能

  • 索引 (index) ：允许多种快速查询，且提供 唯一索引、复合索引、地理空间索引和全文索引 #TODO：了解全文索引？
  • 聚合 (aggregation) ：支持 聚合管道 (aggregation pipeline)
  • 特殊的集合类型：支持 存在时间有限 的集合，适用于那些将在某个时间过期的数据，如会话 (session) ;也支持固定大小的集合，用于存放日志
  • 文件存储 (file storage) ：用于存储大文件和文件元数据

MongoDB 不支持关系型数据库中的 join 和复杂的 多行事务 (multirow transaction)

卓越的性能

MongoDB 能对文档进行 动态填充 (dynamic padding)，也能 预先分配数据文件 以利用额外的空间来换取稳定的性能

MongoDB将尽可能多的内存用作 缓存Cache，试图为每次查询 自动选择 正确的索引
