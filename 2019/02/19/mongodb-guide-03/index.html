MongoDB 03 创建、更新和删除文档

内容

  • 删除文档 remove、drop

插入并保存文档

批量插入 insert

如果没有指定 _id ，全自动生成一个

如果向集合中插入多个文档，使用批量插入会快点，可以将一组文档传递给数据库

删除文档 remove、drop

remove 函数接受一个查询文档作为可选参数，给定这个参数以后，只有符合条件的文档才被删除，删除数据是永久性，不能撤销，也不能恢复

drop 在不指定限定文档的条件下，将删除整个集合及其元数据

更新文档

将 date 和 expired_date 进行组合成 relation_date ，并将原来的替换，可以在 shell 中实现

关键：确保指定一个唯一的 "_id"

使用修改器

可以使用原子性的更新修改器(update modifier)，指定对文档中的某些字段进行更新。更新修改器是种特殊的键

$set 修改器

$set 用于指定一个字段的值，如果这个字段不存在，则创建它。可用于修改或增加字段

为上述中的task增加一个operator字段，并设置为J4K

使用 $set 修改内嵌文档，用 field1.field2，如下，将 relation_date.date 更新当前时间

$inc 增加或减少

$inc 修改器用来增加已有键的值，或者该键不存在那就创建一个

$inc只能用于整型、长整型或双精度浮点型的值，其值必须为数字，指定负数即减少

假设上述的任务已经完成了30%，使用 $inc 就可以完成操作

$push $each 数组修改器

$push 会向已有的数组末尾加入一个元素，要是没有就创建一个新的数组

对任务 task 进行评价，可以有多个评价，新增comments字段

$each子操作可以通过$push向数组添加多个值

$slice子操作可以用于控制数组长度，只能是负数

删除元素

使用 $pop 修改器可以从数组的任何一端删除元素，{"$pop": {"key": 1}} 从数组的末尾删除，{"$pop": {"key": -1}} 从数组的头部删除

使用 $pull 删除数组中匹配的元素

基于位置的数组修改器

有两种方法操作数组中的值：通过位置或定位操作符$

直接使用数组下标进行修改，将上述的成员J4A修改成J4B，使用下标0

加入新成员J4C后，希望通过位置操作符$将其修改为J4D

修改器的速度

数组修改器可能会改变文档的大小，就会慢一些，因为要移动文档

upsert

update 的第三个参数

upsert是一种特殊的更新，要是没有找到符合更新条件的文档，就会以这个条件和更新文档为基础创建一个新的文档

同一套代码既可以用于创建文档又可以用于更新文档

使用upsert的方法，就是指定 update 方法的第三个参数，将其设置为 true

更新多个文档

update 的第四个参数

默认情况下，更新只能对符合匹配条件的第一个文档执行操作，要指明对多个文档进行修改，可将 update 的第四个参数设置为 true

返回被更新的文档

findAndModify命令能够在一个操作中返回匹配结果并且进行更新
