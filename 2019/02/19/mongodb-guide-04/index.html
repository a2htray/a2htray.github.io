MongoDB 查询操作

find 简介

find 的第一个参数决定了要返回哪些文档，这个参数是一个文档，用于指定查询条件。空 {} 的查询条件，将会返回全部内容，默认为空

查询文档中的多个 键值对 表示查询的与关系，即 条件1 and 条件2 and ...

指定要返回的键

find 的第二个参数指定要返回的键，可用于减少数据传输，如用户 user 集合，只需要用户的 email 和 nickname：

限制

使用查询时，会有一些限制，即不能引用文档其他键的值，在限定文档中不能使用 this 关键字

查询条件

$lt、$lte、$gt 和 $gte 就是全部的比较操作符，分别对应 <、<=、>、>=

$ne 就是 not equal，即 !=

$in 属于数组中的值，返回与数组中值相匹配的文档

$nin 不属于数组中的值，返回与数组中值不匹配的文档

$in 和 $nin 的数组中的元素可以是不同类型

$or 接收一个包含所有条件的数组

$not 是元条件句，即可以用在任何其他条件上

条件语句是内嵌文档的键，修改器是外嵌文档的键，对于一个键来说，可以有多个条件语句，但只能有一个修改器，如下面的修改器是非法的：

特定类型的查询

null 类型

正则表达式

使用 //，进行正则匹配

查询数组

查询数组元素和查询标量值一样

$all 可用于多个元素来匹配数组，数组中的顺序无关紧要

利用下标查询，使用 key.index

$size：用于查询特定长度的数组

$slice：可以返回匹配文档中数组的子集

也可以指定一个数组，数组的第二个元素表明要取的元素个数

查询内嵌文档

设有文档

$where 查询

使用 $where 子句，执行任意 JavaScript 程序

游标

可以限制结果的数量，根据任意键按任意顺序组合对结果进行各种排序，或者是执行其他一些强大的操作

数据库用游标返回find的执行结果

limit、skip、sort

定义了多个函数用于对数据的筛选

避免 skip 跳过大量数据

使用 skip 会先找到匹配的数据，在从中进行筛选

随机选取文档

随机选取文档的小技巧，在插入文档时，每个文档带一个随机数。在查询时，通过比较随机数大小，只返回比随机数大或比随机数小的一个文档，即完成了随机选取文档的操作

高级查询选项

MongoDB 的查询分成两类：简单查询 (plain query) 和封装查询 (wrapped query)

封装查询即再对查询语句加入必要的限制，以达到特定的目标

  • $maxscan：指定本次查询中扫描文档数量的上限
  • $showDiskLoc：用于显示文档在磁盘中的位置

一些操作视具体驱动器而定
